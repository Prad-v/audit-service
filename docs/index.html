Audit Service Documentation

Live Demo: Frontend UI | API Docs | Health Check

The Audit Service is a comprehensive audit logging system that provides secure, scalable, and real-time audit trail capabilities for your applications.

Secure

Role-based access control, encryption, and secure audit trails

Scalable

Built for high-performance with PostgreSQL, Redis, and NATS

Real-time

Instant event streaming and real-time monitoring

Quick Start

1. Start the Services

# Start all services
./scripts/start.sh start

# Or start in development mode
./scripts/start.sh dev

2. Create Your First Audit Event

curl -X POST http://localhost:8000/api/v1/audit/events \
  -H "Content-Type: application/json" \
  -d '{
    "event_type": "user_login",
    "action": "login",
    "status": "success",
    "tenant_id": "demo_tenant",
    "service_name": "demo_service",
    "user_id": "user_123"
  }'

3. Query Audit Events

# Get all events
curl "http://localhost:8000/api/v1/audit/events?page=1&size=10"

# Filter by event type
curl "http://localhost:8000/api/v1/audit/events?event_types=user_login"

Create Audit Event

POST /api/v1/audit/events

Create a single audit event.

{
  "event_type": "user_login",
  "action": "login",
  "status": "success",
  "tenant_id": "tenant_123",
  "service_name": "auth_service",
  "user_id": "user_456"
}

Query Audit Events

GET /api/v1/audit/events

Query audit events with filtering, sorting, and pagination.

GET /api/v1/audit/events?page=1&size=10&event_types=user_login&status=success

Authentication

JWT Tokens

Authorization: Bearer <your-jwt-token>

API Keys

X-API-Key: <your-api-key>

Development Mode: For development and testing, RBAC can be disabled using environment variables: RBAC_AUTHENTICATION_DISABLED=true and RBAC_AUTHORIZATION_DISABLED=true

Integration Examples

Python Example

import requests

response = requests.post(
    "http://localhost:8000/api/v1/audit/events",
    json={
        "event_type": "user_login",
        "action": "login",
        "status": "success",
        "tenant_id": "demo_tenant",
        "service_name": "demo_service",
        "user_id": "user_123"
    }
)

print(f"Event created: {response.json()['audit_id']}")

JavaScript Example

const response = await fetch('http://localhost:8000/api/v1/audit/events', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    event_type: 'user_login',
    action: 'login',
    status: 'success',
    tenant_id: 'demo_tenant',
    service_name: 'demo_service',
    user_id: 'user_123'
  })
});

const result = await response.json();
console.log(`Event created: ${result.audit_id}`);

Deployment

Docker Compose

# Production deployment
docker-compose up -d

# Development deployment
docker-compose -f docker-compose.yml -f docker-compose.dev.yml up -d

Environment Variables

DATABASE_URL=postgresql://user:password@localhost:5432/audit_db
REDIS_URL=redis://localhost:6379/0
NATS_URL=nats://localhost:4222
RBAC_AUTHENTICATION_DISABLED=false
RBAC_AUTHORIZATION_DISABLED=false

Testing

Run Test Suite

# Run all tests
python3 run_tests.py

# Run specific test types
python3 run_tests.py --unit
python3 run_tests.py --integration
python3 run_tests.py --e2e

# Run with continue on fail
python3 run_tests.py --continue-on-fail

Manual Testing

# Test API health
curl http://localhost:8000/health/

# Test creating an event
curl -X POST http://localhost:8000/api/v1/audit/events \
  -H "Content-Type: application/json" \
  -d '{"event_type": "test", "action": "test", "status": "success", "service_name": "test"}'

Troubleshooting

Services Not Starting

# Check Docker status
docker ps

# Check logs
docker-compose logs

# Restart services
docker-compose restart

Database Connection Issues

# Check database logs
docker-compose logs postgres

# Restart database
docker-compose restart postgres

API Not Responding

# Check API logs
docker-compose logs api

# Restart API
docker-compose restart api
