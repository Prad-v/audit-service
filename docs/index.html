The Audit Service is a comprehensive audit logging system that provides secure, scalable, and real-time audit trail capabilities for your applications. It includes a powerful alerting system for real-time event monitoring and notification.

Secure

Role-based access control, encryption, and secure audit trails

Scalable

Built for high-performance with PostgreSQL, Redis, and NATS

Real-time

Instant event streaming and real-time monitoring

Quick Start

1. Start the Services

# Start all services
./scripts/start.sh start

# Or start in development mode
./scripts/start.sh dev

2. Create Your First Audit Event

curl -X POST http://localhost:8000/api/v1/audit/events \
  -H "Content-Type: application/json" \
  -d '{
    "event_type": "user_login",
    "action": "login",
    "status": "success",
    "tenant_id": "demo_tenant",
    "service_name": "demo_service",
    "user_id": "user_123"
  }'

3. Query Audit Events

# Get all events
curl "http://localhost:8000/api/v1/audit/events?page=1&size=10"

# Filter by event type
curl "http://localhost:8000/api/v1/audit/events?event_types=user_login"

Integration Examples

Python Example

import requests

response = requests.post(
    "http://localhost:8000/api/v1/audit/events",
    json={
        "event_type": "user_login",
        "action": "login",
        "status": "success",
        "tenant_id": "demo_tenant",
        "service_name": "demo_service",
        "user_id": "user_123"
    }
)

print(f"Event created: {response.json()['audit_id']}")

JavaScript Example

const response = await fetch('http://localhost:8000/api/v1/audit/events', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    event_type: 'user_login',
    action: 'login',
    status: 'success',
    tenant_id: 'demo_tenant',
    service_name: 'demo_service',
    user_id: 'user_123'
  })
});

const result = await response.json();
console.log(`Event created: ${result.audit_id}`);

Alerting System

Alerting Service: Alerting API Docs | Health Check

The Alerting Service provides real-time, policy-based alerting capabilities for audit events. It supports multiple alert providers including PagerDuty, Slack, Webhook, and Email.

Policy-Based Alerting

Define flexible alert policies with complex rule matching and multiple operators

Multiple Providers

Support for PagerDuty, Slack, Webhook, and Email alert delivery

Throttling & Suppression

Prevent alert fatigue with configurable throttling and suppression rules

Scalable Architecture

Horizontal scaling support with async processing and delivery tracking

Quick Start with Alerting

# Start the alerting service
docker-compose up -d alerting

# Create a webhook provider
curl -X POST http://localhost:8001/api/v1/alerts/providers \
  -H "Authorization: Bearer test-token" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Test Webhook",
    "provider_type": "webhook",
    "config": {
      "url": "http://localhost:8080/webhook",
      "method": "POST",
      "headers": {"Content-Type": "application/json"}
    }
  }'

# Create an alert policy
curl -X POST http://localhost:8001/api/v1/alerts/policies \
  -H "Authorization: Bearer test-token" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Failed Login Alerts",
    "rules": [
      {"field": "event_type", "operator": "eq", "value": "user_login"},
      {"field": "status", "operator": "eq", "value": "failed"}
    ],
    "severity": "high",
    "message_template": "Failed login by {user_id} from {ip_address}",
    "providers": ["webhook-001"]
  }'

# Process an event to trigger alerts
curl -X POST http://localhost:8001/api/v1/alerts/process-event \
  -H "Authorization: Bearer test-token" \
  -H "Content-Type: application/json" \
  -d '{
    "event_type": "user_login",
    "user_id": "john.doe",
    "ip_address": "192.168.1.100",
    "status": "failed"
  }'

Alert Policy Configuration

Alert policies support flexible rule matching with multiple operators:

{
  "name": "Security Alert Policy",
  "description": "Alert on security-related events",
  "enabled": true,
  "rules": [
    {
      "field": "event_type",
      "operator": "in",
      "value": ["user_login", "permission_change", "data_access"],
      "case_sensitive": true
    },
    {
      "field": "severity",
      "operator": "gte",
      "value": "high",
      "case_sensitive": false
    }
  ],
  "match_all": true,
  "severity": "critical",
  "message_template": "Security alert: {event_type} by {user_id}",
  "summary_template": "Security event detected",
  "throttle_minutes": 5,
  "max_alerts_per_hour": 10,
  "providers": ["pagerduty-001", "slack-001"]
}
